### ✏️ 키워드 정리

- 객체를 복사하는 방법은 Object.assign 정적 메서드나 스프레드 구문을 사용하는 방법이 있다.

- 객체의 최상위 속성만 복사하는 것을 얉은 복사라고 하며, 객체의 속성에 중첩된 객체가 있는 경우는 여전히 원본 객체에도 영향을 미친다. 최상위 속성뿐만 아니라 모든 속성에 참조가 전혀 없는 완전한 복사를 깊은 복사라고 한다.

- 특정 작업을 반복하거나 순회하는 경우 자바스크립트에서는 이터러블 프로토콜과 이터레이터 프로토콜이라는 규칙을 사용한다.

- Map 객체는 new 키워드로 생성한다. set 메서드로 키와 값의 쌍을 추가하며, has메서드로 존재여부를 확인하고, delete 메서드로 키와값의 쌍을 삭제하고, get 메서드와 같이 키로 값을 조회할 수 있다.

- Set 객체도 역시 new 키워드로 생성한다. 다만 Map 객체와 달리 값만 저장한다. 중복된 값을 가지지 않기 때문에 유일한 값만을 포함해야 하는 자료구조에 적합하다. 실제로 교집합, 합집합과 같은 다양한 집합 연산에 쓸 수 있다.

## ✨ 딱 하나만 기억한다면? 코드 조각이나 이론 중 가장 중요했던 핵심 포인트 하나만 기록합니다.

- 객체 타입은 변수가 메모리 공간의 주소를 참조하기 때문에 참조타입이라고 불리며, 객체를 다른 변수에 할당하는 경우 객체 자체가 복사되는 것이 아니라 객체를 참조하는 주소가 할당된 것이기 때문에, 객체의 수정이 다른 변수의 객체에도 동일하게 적용된다.

- 자바스크립트의 객체가 생성되면 메모리 공간에 객체와 관련된 데이터가 할당된다. 객체가 많아질수록 더 많은 메모리 공간이 필요하므로 더 이상 사용하지 않는 객체를 정리하여 메모리 공간을 확보하는 과정이 필요하다. 이를 가비지 컬렉션이라고 부르며, 자바스크립트 엔진이 자동으로 수행한다.
