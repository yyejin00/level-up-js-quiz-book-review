## 📌 오늘의 주제 : 정적 타입 vs 동적 타입

## 🔥 한 줄 핵심

- JavaScript는 동적 타입 언어지만 TypeScript를 사용하면 정적 타입처럼 개발할 수 있다.

### ✏️ 키워드 정리

| 기준                    | 정적 타입 언어 | 동적 타입 언어                     |
| ----------------------- | -------------- | ---------------------------------- |
| **타입 결정 시점**      | 컴파일 시      | 실행 시                            |
| **에러 발견 시점**      | 컴파일 단계    | 런타임                             |
| **타입 변경 가능 여부** | 거의 불가능    | 자유롭게 변경                      |
| **성능**                | 보통 더 빠름   | 런타임 검사 때문에 상대적으로 느림 |
| **코드 안정성**         | 높음           | 낮음                               |
| **개발 속도**           | 초기 속도 느림 | 빠름                               |
| **대규모 프로젝트**     | 매우 유리      | 관리 어려움                        |

- 추가 정리 내용 : https://www.notion.so/youngbase/TS-VS-JS-320fb8f030b080c7b9a1f09569c4c259

### 결론

- TypeScript는 개발할 때만 타입을 검사하고, 실행할 때는 JavaScript로 변환되어 동작한다.
