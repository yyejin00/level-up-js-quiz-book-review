## 📌 오늘의 주제 : Symbol의 이해

## 🔥 한 줄 핵심

- Symbol은 고유하고 변경 불가능한 값을 생성하는 JavaScript의 원시 타입으로, 주로 객체 프로퍼티의 네임스페이스 충돌을 방지하기 위해 사용된다.

### ✏️ Symbol이 추가된 이유

1. 객체 property 충돌 방지
2. 라이브러리 내부 key 보호
3. JavaScript 내부 기능 확장

```js
const user = {
  id: 1,
};

// 다른 코드
user.id = 'abc';

//결과
user.id; // "abc"
```

### 🤖 Symbol 사용법

```js
const id = Symbol("id");

const user = {
  name: "Tom",
  [id]: 123,
  id: 123
}

user.id = 345;

user; //{name: 'Tom', id: 345, Symbol(id): 123}
Object.keys(user); //['name', 'id']
Object.getOwnPropertyNames(user); //['name', 'id']
Object.getOwnPropertySymbols(user); //[Symbol(id)]
Reflect.ownKeys(user); //['name', 'id', Symbol(id)]

```

### ❇️ 관련 메서드 정리
| 메서드 | 가져오는 것 |
|---|---|
| Object.keys() | 문자열 key |
| Object.getOwnPropertyNames() | 모든 문자열 key |
| Object.getOwnPropertySymbols() | Symbol key |
| Reflect.ownKeys() | 문자열 + Symbol 모두 |

### 결론
- 네임스페이스 방지를 위해 개발된 만큼, 코드 구현때 사용해보쟈!
